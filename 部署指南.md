# GitHub 部署指南（零基础版）

## 📋 准备工作

你需要的文件：
- ✅ update-log.html（管理页面）
- ✅ view.html（查看页面）
- ✅ README.md（项目说明）

## 🚀 部署步骤

### 第一步：注册 GitHub 账号

1. 打开浏览器，访问：https://github.com
2. 点击右上角的"Sign up"（注册）按钮
3. 填写邮箱、密码、用户名
4. 完成邮箱验证
5. 登录你的 GitHub 账号

---

### 第二步：创建新仓库

1. 登录后，点击右上角的"+"号，选择"New repository"（新建仓库）
2. 填写仓库信息：
   - **Repository name**（仓库名称）：填写 `update-log`（或其他你喜欢的名字）
   - **Description**（描述）：填写"更新日志系统"
   - **Public**（公开）：选择这个选项（免费用户只能用 Public）
   - ✅ **勾选"Add a README file"**
3. 点击底部绿色按钮"Create repository"（创建仓库）

---

### 第三步：上传文件

#### 方法一：网页直接上传（最简单，推荐！）

1. 在你刚创建的仓库页面，点击"Add file" > "Upload files"
2. 把下面三个文件拖拽到网页中：
   - `update-log.html`
   - `view.html`
   - `README.md`
3. 在底部的"Commit changes"框中：
   - 输入提交说明：`上传更新日志系统文件`
4. 点击绿色按钮"Commit changes"

#### 方法二：使用 GitHub Desktop（可选）

1. 下载 GitHub Desktop：https://desktop.github.com/
2. 安装并登录你的 GitHub 账号
3. 克隆你刚创建的仓库到本地
4. 把文件复制到仓库文件夹
5. 提交并推送更改

---

### 第四步：启用 GitHub Pages（让网站上线）

1. 在仓库页面，点击顶部的"Settings"（设置）
2. 在左侧菜单中找到"Pages"，点击进入
3. 在"Source"（来源）部分：
   - Branch：选择"main"（或"master"）
   - Folder：选择"/ (root)"
4. 点击"Save"（保存）
5. 等待 1-2 分钟，页面会显示你的网站地址

**你的网站地址格式：**
```
https://你的用户名.github.io/update-log/
```

---

### 第五步：访问你的网站

1. 管理页面（添加日志）：
   ```
   https://你的用户名.github.io/update-log/update-log.html
   ```

2. 查看页面（浏览日志）：
   ```
   https://你的用户名.github.io/update-log/view.html
   ```

---

## 🎯 快速访问设置

### 方法一：设置主页为 view.html

如果你想让访问 `https://你的用户名.github.io/update-log/` 时直接显示查看页面：

1. 把 `view.html` 重命名为 `index.html`
2. 重新上传到 GitHub
3. 现在访问 `https://你的用户名.github.io/update-log/` 就会自动显示查看页面

### 方法二：创建导航页

创建一个 `index.html` 文件作为导航页，让用户选择进入哪个页面。

## ⚠️ 重要提示

### 1. 数据存储问题
- 数据存储在**访问者的浏览器**中，不同的人访问会看到**不同的数据**
- 每个人的浏览器都是独立的，A电脑添加的日志，B电脑看不到
- 如果需要多人共享数据，需要手动导出Excel文件分享

### 2. 适用场景
✅ **适合：**
- 个人使用的更新日志
- 团队内部通过 Excel 导入导出共享数据
- 临时展示更新记录

❌ **不适合：**
- 需要多人实时同步的场景（需要后端数据库）
- 大量用户访问的公开网站

### 3. 数据共享方案
如果需要让别人看到你添加的日志：
1. 在管理页面添加日志
2. 点击"导出Excel"
3. 把 Excel 文件发给对方
4. 对方在他们的浏览器中点击"导入Excel"

## 🔧 常见问题

### Q1: 网站打开后看不到数据？
**A:** 这是正常的！第一次打开是空的，你需要：
- 管理页面：点击"添加更新日志"添加数据
- 查看页面：需要先点击"加载数据文件"导入 JSON/Excel

### Q2: 如何修改网站？
**A:**
1. 在本地电脑修改 HTML 文件
2. 重新上传到 GitHub 覆盖旧文件
3. 等待 1-2 分钟，刷新网页即可看到更改

### Q3: 网站地址太长怎么办？
**A:**
1. 可以使用短链接服务（如 bit.ly）
2. 或者购买自己的域名绑定到 GitHub Pages

### Q4: 可以设置密码保护吗？
**A:** GitHub Pages 本身不支持密码保护，如需加密：
1. 可以在 HTML 中添加简单的 JavaScript 密码验证
2. 或者使用第三方服务（如 Cloudflare）
